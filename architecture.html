Core Scheduling Algorithm
Log Aging Formula
Effective Score = Sb + α · ln(1 + twait)
Sb = XGBoost Base Severity Score (0–100)
α = Aging factor (optimal = 0.5–1.0, app uses 8.0)
t = Accumulated wait time in minutes
Key Data Flows

● Patient Admission Flow

  • Clinician submits 26 vitals via Dashboard form
  • POST /api/predict → XGBoost inference (microseconds)
  • SHAP top-5 explanations computed concurrently
  • Patient record appended to WAITING_QUEUE (O(1))
  • All 3 queues re-sorted and returned in response

● Dynamic Priority Recalculation

  • Every GET /api/queue call triggers get_all_queues()
  • Logarithmic age applied to each patient in O(N)
  • Timsort produces final priority order in O(N log N)
  • Reassessment daemon sets needs_reassessment=True
  • Nurse re-submits vitals → score recalculated live

● Doctor Allocation Flow

  • Clinician clicks "Treat" on highest-priority patient
  • POST /api/treat/:id → AVAILABLE_DOCTORS decremented
  • Patient removed from queue, added to BUSY_DOCTORS
  • Doctor Release Daemon checks every 10 s
  • After 15 min, doctor freed → semaphore incremented

● Simulation & Evaluation

  • POST /api/simulate samples 25 real MIMIC-IV patients
  • Background thread adds 1 patient/60 s to mimic arrivals
  • 94-patient 8-hr cohort tested across FIFO / Priority / Aging
  • Alpha sensitivity analysis over α = 0.0 → 5.0
  • /api/fairness checks AUROC across age/gender strata
